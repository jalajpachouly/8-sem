Hyperparameters in Neural Networks

Purpose, types, and tuning methods — with diagrams

What are Hyperparameters?

Hyperparameters are configuration settings for a neural network that are set before training begins — unlike model parameters (weights & biases) which are learned during training. They control the learning process and define the model's architecture.

Proper tuning is essential: wrong hyperparameters cause underfitting (model too simple) or overfitting (model memorises training data).

Methods for Tuning Hyperparameters

Figure 3 — Hyperparameter Tuning Method Comparison
Six tuning strategies — from manual trial-and-error to intelligent Bayesian optimisation and resource-efficient Hyperband.

1. Manual Search

User sets values based on intuition and experience.

✔ Easy, no extra tools

✗ Time-consuming, may miss optimum

2. Grid Search

Tests all combinations within a predefined range.

✔ Exhaustive — guaranteed to find best in range

✗ Computationally expensive; infeasible for large spaces

3. Random Search

Randomly samples within the hyperparameter range.

✔ More efficient than grid for high-dimensional spaces

✗ May miss optimum due to randomness

4. Bayesian Optimization

Uses probabilistic models to focus on promising regions.

✔ Efficient; focuses search intelligently

✗ Complex to implement; expensive overhead

5. Gradient-Based Optimization

Treats hyperparameters as continuous variables; uses gradients.

✔ Fast for differentiable hyperparameters

✗ Requires differentiability; local minima risk

6. Evolutionary Algorithms

Mimics natural selection to evolve optimal configurations.

✔ Handles non-differentiable, complex spaces

✗ Expensive; the algorithm itself needs tuning
